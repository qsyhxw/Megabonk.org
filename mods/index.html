Megabonk Mods & Modding Guide

Choose the right manager, install a clean profile, compare active mods, check game-version compatibility, and recover safely when a package breaks.

Game reference: v1.0.69 Catalog checked: July 24, 2026 129 Thunderstore packages

Does Megabonk support mods?

Community mods are available, but they are not an official built-in feature. Megabonk's Steam listing does not advertise a mod browser or Workshop integration. The organized community catalog is on Thunderstore; the easiest setup is a separate profile in r2modman or Gale, with BepInEx IL2CPP installed as a declared dependency.

Manager, loader, or mod?

Manager

r2modman or Gale

Creates profiles, downloads packages, resolves declared dependencies, toggles mods, and launches a modded or vanilla session.

Loader

BepInEx IL2CPP

Loads compatible code into this Unity IL2CPP game. Most BepInEx mods declare the required pack version automatically.

Content

Individual mods

Add UI, characters, co-op, automation, or gameplay changes. Each package has its own dependencies and patch risk.

Searching for “Megabonk Nexus”? Use the original Thunderstore package or the author-linked GitHub repository when available. We did not find an official Nexus destination linked by the Steam listing or the selected package authors, so avoid treating a search-result reupload as authoritative.

How to Install Megabonk Mods

This manager-first route keeps mod files in a named profile and makes rollback much easier than copying random files into the game directory.

Close the game and back up

Exit Megabonk completely. Copy the whole save folder to a dated backup before changing the loader, profile, or save-affecting mods.

Install one manager

Get r2modman or Gale from its original page. Do not run an installer from a repost.

Select Megabonk

Create a new profile named for the game version or purpose, such as “1.0.69 UI test.” Keep a clean profile for comparison.

Install one package

Choose the Mod Manager install option. Accept the package's declared dependencies instead of downloading a random BepInEx build manually.

Launch modded

Start through the manager's modded launch button. Confirm the title screen loads, then run a short low-stakes test before opening an important save.

Add mods one at a time

After each successful test, add the next package. This makes a bad version, missing dependency, or conflict easy to isolate.

Best baseline: one clean profile, one test profile, and one dated save backup. A profile name should record the game version and purpose rather than “new profile.”

How to Check Mod Compatibility

A recent package date is useful, but it is not proof. Check the whole chain before launching: game build, manager, loader, dependencies, mod, and other enabled packages.

1. Game version

Record the current Megabonk version before updating or troubleshooting.

2. Package record

Read the package version, update date, changelog, and pinned instructions.

3. Dependencies

Check the exact BepInEx or library versions declared by that package.

4. Recent reports

Look for current issues mentioning your game and package versions.

StatusSignalsWhat to do
Ready to testRecent update or explicit current-version note; dependencies resolve; no current blocking issue.Test it alone in a separate profile and disposable run.
UnclearOlder package, no game-version claim, or mixed recent reports.Keep it disabled on the main profile; wait or test only after a full backup.
Do not use yetDeprecated package, unresolved loader error, missing dependency, crash loop, or save corruption report.Remove it from the profile and use the last known-good version only if the author documents that rollback.

After a Megabonk update: launch vanilla first. Do not update the game, loader, and every mod simultaneously and then try to diagnose the result.

Back Up Your Save Before Modding

  1. Exit Megabonk and the mod manager completely.
  2. Open %USERPROFILE%\AppData\LocalLow\ and locate the active Megabonk save directory. The final developer and save-folder names can vary, so confirm the folder from its recent files before copying it.
  3. Copy the entire Saves folder to a separate dated folder.
  4. Keep at least one backup made before installing any save editor, unlocker, item spawner, or multiplayer mod.
  5. Do not edit the only copy of a save in place.

Steam Cloud is useful, but synchronization is not the same as a versioned manual backup. If a modified or corrupted save syncs, the cloud copy may also become the wrong copy. See the Megabonk Save Recovery Guide before restoring files.

How to Disable or Uninstall Mods

Disable the last mod

Turn off the most recently added package, keep its required libraries in place, and test the profile again.

Disable all mods

If the problem remains, disable the complete profile or switch to the clean profile. This separates game problems from mod problems.

Verify Steam files

If vanilla still fails, use Steam's installed-files verification. This restores missing or changed game files without asking you to delete saves.

Restore only when needed

Restore a dated save only if the problem follows the save into a verified clean game. Preserve the current copy before overwriting it.

Do not troubleshoot by deleting folders at random. A manager profile, the game installation, and the save folder are different layers. Identify which layer is broken before removing anything.

Mod Safety and Fair-Play Boundaries

  • Third-party code can access your user account's files and network. Use original package pages, inspect linked source repositories when available, and avoid unknown executables or password-protected archives.
  • Do not submit gameplay-changing modded runs to official leaderboards. Unlockers, item spawners, stat changes, automation, custom characters, and netplay make results incomparable with vanilla runs.
  • Use matching multiplayer profiles. Different mod or dependency versions can cause desync, missing objects, or failed connections.
  • Keep private data out of support logs. Review logs before posting them publicly and remove usernames, local paths, tokens, or invite details when present.
  • Do not assume “popular” means harmless. Download count measures adoption, not security review or compatibility with your current setup.

Stop immediately if a package asks you to disable antivirus globally, enter a Steam password outside Steam, run an unrelated administrator script, or download dependencies from an unlisted mirror.

Megabonk Mods FAQ

Is the Megabonk Toggler a mod?

No. Toggler is a native Shop Upgrade unlocked after purchasing 40 unlocks. It needs no download, manager, shortcut, or config file. Use the Toggler unlock and usage guide.

Does Megabonk officially support mods?

No built-in mod browser or Steam Workshop feature is listed for the game. Current mods are community projects and are used at your own risk.

What is the easiest way to mod Megabonk?

Install r2modman or Gale from its original page, select Megabonk, create a separate profile, and install one package through the manager so declared BepInEx dependencies are included.

Should I use Nexus or Thunderstore for Megabonk mods?

Thunderstore is the organized catalog used by the current Megabonk packages in this guide. Prefer original package pages and author-linked source repositories rather than an unverified reupload found through search.

Are Megabonk mods safe?

No third-party code is guaranteed safe. Back up saves, check the author and package history, use original sources, scan downloads, and avoid unknown installers. Keep gameplay-changing mods away from official leaderboard runs.

Can mods add multiplayer?

Yes, community projects such as MegabonkTogether and BonkWithFriends add unofficial netplay or Steamworks P2P co-op. They do not change the vanilla game's official single-player status and can have patch or synchronization problems.

Why did my mods stop working after an update?

The update may have changed the game code expected by BepInEx, a library, or the mod itself. Launch vanilla, compare package and dependency versions, then re-enable one current mod at a time.

How do I return to vanilla Megabonk?

Switch to a clean manager profile or disable the modded profile. If vanilla still fails, verify installed files in Steam. Restore a save backup only if the clean verified game still has a save-specific problem.
